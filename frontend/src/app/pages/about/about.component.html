Rules

The rules are simple. Don't break the webapp! besides you should follow along the following guidelines:

  • Don't use any automated tools to solve the challenges.
  • Don't use AI, as this challenges should refresh your know-how.
  • Don't share the solutions to other participants.
  • Every challenge related information is in the challenge page, there are no hidden information elsewhere (e.g. in the webapp source code)

About the Webapp

The source code of the webapp can be found on Github. The webapp is written in JavaScript (Angular, TS), uses a FastAPI (Python) backend and uses a PostgreSQL database. In addition to store the challenges, a MongoDB is used.

As I like to exaggerate a bit, in addition to the stack above, the webapp is deployed on a Kubernetes cluster and uses a CI/CD pipeline to deploy the changes to the cluster. The cluster is hosted privately.
